# Hey Milhao — Plan (하네스 2단계)

> **What pages, components, and build phases exist.**
> Read with REQUIREMENTS.md (scope) and PRD.md (specs).

## Pages / Routes

| Route | Purpose |
|---|---|
| `/` | Author profile + latest post list |
| `/posts/[slug]` | Individual post |
| `/about` | Static about page |
| `/privacy` | Privacy policy (required for AdSense) |
| `/api/views/[slug]` | View counter (GET reads, POST increments) |
| `/sitemap.xml` | Auto-generated by next-sitemap |
| `/robots.txt` | Static, allow all + sitemap reference |

## Components (high level)

- `Hero` — author profile card on home
- `PostCard` — single post preview tile
- `PostList` — wrapper that maps posts → cards
- `PostHeader` — title / byline / date / view count inside post page
- `MDXContent` — wraps MDX with prose styling
- `ViewCounter` — client component, fetches `/api/views/[slug]`
- `AdSlot` — wraps `<ins class="adsbygoogle">`. Variants: header / in-article / footer
- `SubscribeButton` — disabled placeholder with "Coming soon" tooltip
- `Footer` — copyright + links

## Build Phases (code phase, runs after docs are approved)

### Phase A — Skeleton
1. `create-next-app` (TS, Tailwind, App Router, ESLint)
2. Install deps: `@next/mdx`, `gray-matter`, `@upstash/redis`, `next-sitemap`, `@tailwindcss/typography`
3. Tailwind theme tokens (colors / fonts) tuned to maily.so/josh
4. Root layout: nav + footer + base typography
5. Empty stubs for `/`, `/posts/[slug]`, one dummy MDX

### Phase B — Content & Visual Parity
6. Copy 5–8 maily.so/josh posts as placeholder MDX (frontmatter + body)
7. Build `Hero` matching reference
8. Build `PostCard` + `PostList`
9. Build post page template with `MDXContent`
10. Wire `AdSlot` placeholders (no real publisher ID yet)

### Phase C — Counters + SEO
11. `lib/redis.ts` Upstash client
12. `/api/views/[slug]/route.ts` (GET + POST)
13. `ViewCounter` component (client)
14. Open Graph metadata generators
15. `next-sitemap.config.js` + run on build

### Phase D — Polish & Verify (하네스 6단계 hooks)
16. ESLint clean
17. `tsc --noEmit` clean
18. `npm run build` succeeds
19. Lighthouse run on `next start`; iterate until thresholds met
20. README.md (run + deploy instructions)
21. `.env.example` with required keys

## Environment Variables

```
UPSTASH_REDIS_REST_URL=
UPSTASH_REDIS_REST_TOKEN=
NEXT_PUBLIC_ADSENSE_CLIENT_ID=ca-pub-XXXXXXXXXXXXXXXX
NEXT_PUBLIC_SITE_URL=https://heymilhao.com
```

All `NEXT_PUBLIC_*` vars are safe to expose in client bundle. Upstash keys stay server-side (used only inside `/api/views/...`).

## Deploy

- **Vercel** — connect Git repo, paste env vars, auto-deploy on push to `main`.
- **Domain** — heymilhao.com purchased separately; DNS → Vercel.
- **AdSense submission** — only after replacing placeholder content + custom domain attached + ≥15 original posts.
