Um amigo da Cidade do México me mandou um perfil que genuinamente me surpreendeu. GitHub público. Oito repositórios pinados. Nenhum deles é uma biblioteca. Cada um é um negócio de software completo, deployável e monetizável.

Isso é uma coisa agora.

## O que significa "forkar uma empresa" na prática

Cada um dos oito repositórios contém:

- Uma landing page com copy e brand kit.
- Um produto funcional (tipicamente uma camada fina sobre um foundation model com um workflow real em cima).
- Integração com Stripe, termos de uso, política de privacidade, fluxos de email automatizados.
- Um `runbook.md` que explica como operar o negócio em menos de uma hora por semana.
- Um `forecast.md` com números realistas e conservadores.

Forkar o repositório, trocar algumas variáveis de ambiente e apontar para um novo domínio sobe um negócio competidor ou complementar em mais ou menos um fim de semana.

## Por que isso funciona em 2026

Três mudanças estruturais:

1. **O padrão de landing page de SaaS está resolvido.** Tailwind + alguns componentes + um copy bom geram conversão indistinguível, independentemente de quem construiu.
2. **A camada de auth/billing está commoditizada.** Stripe + provedores estilo Clerk + Postgres-as-a-service comprimem as partes chatas.
3. **O wedge é conteúdo + distribuição, não código.** O repositório é a parte chata. A história que você conta sobre o repositório é o moat.

## O filtro do bom gosto

O padrão só funciona para fundadores com bom gosto. O gargalo não é engenharia. É a habilidade de olhar para um clone meio cru e enviar exatamente os ajustes cosméticos e de copy certos para que o produto pareça real para um cliente real.

Esse filtro de bom gosto é o que vai separar os vencedores e perdedores da onda latina de "dez empresas até o Natal" que está começando agora.

## O que eu testaria

Se você sempre quis começar um pequeno negócio de software mas nunca chegou lá, o custo desabou. Pegue um dos templates "repo-as-a-business" visíveis circulando por aí. Gaste um fim de semana forkando, repintando e shipping. Gaste a segunda escrevendo um post de lançamento real.

O pior caso é você aprender o que não quer fazer pelo resto do ano. O melhor caso é seu projeto de fim de semana pagar seu aluguel até o terceiro trimestre.
